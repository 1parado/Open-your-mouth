# 分阶段实现计划

## 目标

当前产品目标已经明确为：

**一个以“老师角色”为核心、支持实时语音互动，并可逐步升级为数字人老师体系的 AI 口语陪练产品。**

因此实现顺序不再围绕“课程平台”展开，而是围绕：

1. 老师
2. 练习入口
3. 实时对话
4. 教学反馈
5. 数字人呈现

## 当前状态

当前项目已经具备以下基础：

- 用户注册 / 登录
- Web 登录态与基础页面
- Provider Gateway：LLM / ASR / TTS / 发音评测统一接口
- 老师卡片首页
- 基于静态老师配置的老师会话页
- 不同老师的 voice 和 prompt 差异化配置

缺少的核心能力主要是：

- 后端老师数据模型
- 场景与练习会话数据持久化
- 真正的实时语音链路
- 数字人渲染层
- 后台老师配置系统

## Phase 1：老师驱动的可用版本

### 目标

让用户在登录后可以：

- 看到老师卡片
- 进入某位老师的会话页
- 和老师进行语音或文字对话
- 听到该老师的专属语音回复

### 范围

- 首页老师卡片
- 老师详情 / 会话页
- 静态老师配置
- Gateway 聊天 + TTS
- 浏览器语音识别兜底

### 成功标准

- 至少 3 位风格不同的老师
- 每位老师都有不同 prompt 和 voice
- 用户能完成一轮“老师开场 -> 用户说话 -> 老师回复”

## Phase 2：老师系统后端化

### 目标

把当前前端静态老师配置升级为后端可管理的老师系统。

### 需要实现

- `teachers` 表
- `teacher_voice_profiles` 表
- `teacher_avatar_profiles` 表
- `scenarios` 表
- `practice_sessions` 表
- `GET /api/v1/teachers`
- `GET /api/v1/teachers/:id`
- `GET /api/v1/teachers/:id/scenarios`
- `POST /api/v1/practice-sessions`

### 成功标准

- 首页老师卡片由 API 提供
- 老师详情和场景由 API 驱动
- 会话开始时能创建真实 session

## Phase 3：真实语音练习闭环

### 目标

把当前“页面内调用接口”的方式升级为真正的实时口语练习链路。

### 需要实现

- Realtime Orchestrator
- WebSocket 会话管理
- 音频上传 / 流式输入
- ASR -> LLM -> TTS 串联
- 中断控制
- 会话状态缓存

### 成功标准

- 用户可以持续说话而不是一次一句点按钮
- 老师可以连续追问
- 播报时可以被用户打断

## Phase 4：反馈与成长系统

### 目标

让练习结果真正具有“教学价值”，而不是只有聊天体验。

### 需要实现

- 发音评测持久化
- 语法纠错摘要
- 会话结束反馈卡
- 历史记录页升级
- 推荐下一次练习

### 成功标准

- 每次练习都能产出一份轻量反馈
- 用户能查看最近历史与老师维度记录
- 能继续和同一老师练习

## Phase 5：数字人老师

### 目标

把老师从“语音角色”升级为“可见角色”。

### 需要实现

- `digital_human_profile` 模型
- 老师形象资源配置
- 2D 数字人基础渲染
- 嘴型同步
- 情绪标签到表情映射
- 纯语音模式降级能力

### 成功标准

- 至少 1 位老师支持数字人模式
- 回复时可同步展示口型和基础表情
- 渲染失败时可自动回退为语音模式

## Phase 6：后台与运营体系

### 目标

让老师、场景、提示词、声音和形象都能被管理，而不是只能写死在代码里。

### 需要实现

- 老师管理后台
- 场景管理后台
- 语音配置后台
- 形象配置后台
- 会话质检页
- 指标看板

### 成功标准

- 新增一位老师不需要改前端代码
- 可回看会话并调优 prompt / voice / 场景

## 推荐实施顺序

建议严格按以下顺序推进：

1. Phase 1 做体验闭环
2. Phase 2 做数据模型和 API
3. Phase 3 做实时语音架构
4. Phase 4 做反馈系统
5. Phase 5 做数字人
6. Phase 6 做后台和运营

## 不建议提前做的事情

- 复杂课程平台
- 大而全报表中心
- 先做多租户机构版
- 先做 3D 数字人
- 先做复杂内容商城

## 一句话结论

先把“老师卡片 -> 语音对话 -> 专属老师回复”做成稳定主链路，再逐步把老师系统、实时语音、反馈和数字人能力加上去。
