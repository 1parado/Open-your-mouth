# 功能分期清单

## Phase 1：可感知的老师体验

### 面向用户的功能

- 登录
- 老师卡片首页
- 老师详情 / 会话页
- 文字对话
- 基础语音输入
- 老师专属 TTS 回复

### 面向系统的实现

- 前端老师静态配置
- Gateway 聊天与 TTS 复用
- 浏览器语音识别兜底

## Phase 2：老师数据化

### 面向用户的功能

- 动态老师列表
- 老师风格标签
- 推荐场景

### 面向系统的实现

- 老师数据表
- 声音配置表
- 形象配置表
- 场景表
- 老师 API

## Phase 3：练习会话化

### 面向用户的功能

- 创建真实会话
- 继续上次练习
- 练习历史

### 面向系统的实现

- `practice_sessions`
- `practice_turns`
- 历史记录接口

## Phase 4：实时语音化

### 面向用户的功能

- 连续语音对话
- 打断老师
- 更自然的对话节奏

### 面向系统的实现

- WebSocket
- Realtime Orchestrator
- 会话状态缓存
- 中断控制

## Phase 5：反馈教学化

### 面向用户的功能

- 发音分数
- 流利度反馈
- 更自然表达建议
- 历次成长趋势

### 面向系统的实现

- 发音评测任务
- 语法纠错摘要
- 反馈存储模型
- 历史结果页

## Phase 6：数字人可视化

### 面向用户的功能

- 老师形象展示
- 嘴型同步
- 情绪驱动表情
- 纯语音 / 数字人切换

### 面向系统的实现

- `digital_human_profile`
- 渲染提供方接入
- 情绪标签映射
- 失败降级链路

## Phase 7：后台化与商业化

### 面向用户的功能

- 更多老师
- 老师订阅 / 解锁
- 个性化推荐

### 面向系统的实现

- 老师后台
- 场景后台
- 质检后台
- 指标看板
- 支付 / 订阅体系
